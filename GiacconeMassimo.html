Introduction

Progetto GUI per gioco della vita Conway. Possibile soluzione

Requirements

Realizzazione di una pagina HTML che funga da dispostivo di input-ouput evoluto per l’applicazione ConwayLife25 in Java.

Requirement analysis

Requisiti Funzionali

Input

  • Permettere all'utente di impostare lo stato iniziale delle celle (es. tramite click sulla griglia).
  • Fornire pulsanti per inviare comandi: start, stop e exit.

Output

  • Visualizzare la griglia con aggiornamenti in tempo reale durante l'evoluzione della simulazione.
  • Mostrare feedback e messaggi di stato o errori.

Comunicazione Back-end

  • Stabilire una connessione bidirezionale (ad es. tramite AJAX/Fetch o WebSocket) per sincronizzare input e output con l'applicazione Java.

Requisiti Non Funzionali

  • Usabilità: Interfaccia intuitiva e facile da usare.
  • Performance: Aggiornamenti rapidi e latenza minima.
  • Sicurezza: Validazione degli input e comunicazione sicura (uso di HTTPS/WSS).

Problem analysis

Il progetto affronta il problema di creare un'interfaccia per interagire con la simulazione del "Gioco della Vita" implementata in Java (ConwayLife25).

  • Input dell'utente: Consentire la configurazione iniziale della griglia, impostando lo stato (vivo/morto) di ogni cella tramite interazione diretta.
  • Controllo della simulazione: Fornire comandi (start, stop, exit) per gestire l'evoluzione del gioco.
  • Output visivo: Visualizzare in tempo reale lo stato della griglia in base alle regole del gioco.
  • Comunicazione: Stabilire un canale bidirezionale tra la pagina HTML e l'applicazione Java per sincronizzare input e output.
